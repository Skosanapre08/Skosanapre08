Some people fall in love with embedded systems by accident. For me, it happened the moment I realized that behind every "smart" device  every sensor, every microcontroller quietly reading a signal and making a decision  there's a whole hidden layer of engineering that most people never think about. That's the layer I want to spend my career in.

I'm Nosipho Skosana, an aspiring Embedded Systems Engineer studying Computer Systems Engineering at Vaal University of Technology (VUT). My interest in this field started in school, where I first got curious about how hardware and software actually talk to each other. It became a real passion during my internship at Resolution Circle, where I got hands-on exposure to embedded hardware and low-level development — and realized this wasn't just an interest, it was the direction I wanted to build a career around.

Since then, I've been deliberately building a strong foundation:
- Linux fundamentals and command-line tools
- Networking fundamentals (TCP/IP, sockets)
- Serial communication protocols: UART, SPI, I2C, CAN, RS485
- Microcontroller programming

Because I believe the best way to learn embedded systems is to actually build things, I've created several of my own projects outside of coursework:
- A Linux-based UART-to-network gateway that bridges serial devices to TCP networks
- A distributed edge telemetry and real-time traffic controller system, simulating a smart-city network of sensor nodes reporting to a central gateway
- A containerized industrial IIoT gateway with a multi-threaded RS485 sensor node, storing live sensor data and serving it through a real-time dashboard

Each project pushed me to understand not just how these protocols and systems work in theory, but how they behave when you actually build with them — timing issues, edge cases, and all the things a textbook doesn't tell you.
